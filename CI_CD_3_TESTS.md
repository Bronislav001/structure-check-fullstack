# CI/CD: 3 тестовых блока

В проекте настроен workflow `.github/workflows/ci.yml`.

## Что запускается в CI/CD

### 1. Backend tests
- Runner: Ubuntu
- Python: 3.12
- Команда: `pytest -q`
- Проверяет backend: API, авторизацию, роли, CRUD, фильтрацию, файлы, SEO и внешний API.

### 2. Frontend tests
- Runner: Ubuntu
- Node.js: 20
- Команда: `npm run test`
- Проверяет React-компоненты, форму входа, защиту маршрутов, страницу проверки, SEO-компонент и внешний API-блок.

### 3. E2E tests
- Runner: Ubuntu
- Playwright
- Команда: `npm run test:e2e`
- Проверяет пользовательские сценарии через браузер: редирект гостя на вход и корректную обработку ошибки внешнего API.

## Что происходит после тестов

После успешного прохождения всех трёх тестовых блоков запускается job `Docker build after 3 tests`:

- сборка backend Docker image;
- сборка frontend Docker image;
- проверка `docker compose config`.

Если хотя бы один из трёх тестовых блоков падает, Docker-сборка не запускается.

## Что сказать на защите

В CI/CD настроены три обязательных тестовых этапа: backend, frontend и E2E. Docker build запускается только после успешного прохождения всех трёх тестов. Это подтверждает, что перед сборкой контейнеров автоматически проверяются серверная часть, клиентская часть и основные пользовательские сценарии.
